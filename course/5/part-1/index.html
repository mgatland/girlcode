Project setup

Starting on the project

First, clone the project and make sure it works on your computer.

To clone the project, go to the list of Girl Code projects at https://github.com/girlcodeakl/, click on the project for this class, and click this button:

(You will need to be signed in to the Github website.)

Right click on the project name in Github Desktop and choose 'Open in Terminal' or 'Open in Git Shell'

This will open a Terminal or PowerShell window in the right folder.

Now you need to install the Node dependencies. Do this with the command npm install in Terminal\PowerShell.

Next, start the app with this command: node index.js

Your terminal should now be busy running the server.

Go back to your web browser and go to http://localhost:3000/index.html to visit your local server.

It should look a lot like last week's project. We are using that project as a starting point for our app.

Welcome to the workforce

From now on, Girl Code will always start this way:

  • Find a partner and decide who's driving.
  • We'll talk about the new tasks in this week's sprint.
  • Choose a task from the project wall and write your names next to it.
  • Go to the project's issues page, and assign the digital version of the issue to one of you as well.
  • In Github Desktop, click 'Sync' to update and make sure you're in the master branch.
  • Create a new branch for your work. Give it a name based on the task, i.e. "2-make-submit-not-go-to-a-new-page" for issue 2.

Then you can work on the task! Make a commit whenever you feel you've completed a step.

When the job is done, make your final commit then make a Pull Request.Write a short comment that includes the issue number with a hash, i.e. "fixed #2 - submit button now stays on the same page." Tagging the issue number lets Github know it should display that Pull Request when someone looks at that issue.

Get one of the mentors to do a code review and merge your changes. Once it's merged, you can close the issue on Github and move the Post-It note to 'Done' on the project wall.

If there's time, you can pick up another ticket. Otherwise, get ready to describe your change in the end-of-sprint demo.
