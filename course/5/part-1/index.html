Starting the project

Let's get ready to work on the team project.

This page has two sections: first time setup and Starting each week.

First time setup

Logging In

Make sure you have signed in:

  • Sign in to the GitHub website
  • Sign in to GitHub Desktop (an app on your computer). You do this by clicking File, Options, and make sure you have the correct details under Accounts and Git.
  • You might have to close and reopen your web browser after you have signed into GitHub Desktop before everything connects together.

Get access

(You only have to do this the first time.)

Our mentors will send you an invitation to join the new project. Check your email for a message from GitHub. You will have to open the email and click a few links to accept the invitation.

Download the Project

(You only have to do this the first time.)

1. Go to the list of Girl Code projects at https://github.com/girlcodeakl/

2. Your team's project should be on the list (ask which one it is if we haven't said). Click on the project to go to the project page.

3. Click 'Clone or download', then choose 'Open in desktop' to clone the project using Github Desktop.


This will create a clone (a copy) of the project on your computer. Keep a note of where on your computer it was created!

Not working? If this doesn't work, it usually means you're not logged into your GitHub account on Google Chrome, or in GitHub Desktop (the app on your computer). If it still doesn't work, try restarting Chrome and trying again. If it still doesn't work, ask our mentors for help.

Starting each week

We need to get four tools ready:

  • Github Desktop to share our changes
  • Command Prompt or Terminal to run a server
  • Google Chrome (or other web browser) to test our changes
  • Atom to edit our code

Github Desktop

Make sure the right respository is selected.

Command Prompt or Terminal

In Github Desktop, click Repository then Open in Terminal (Mac) or Open in Command Prompt (Windows)

This will open a Terminal or Command Prompt window in the right folder.

If this is the first time, type npm install into Terminal or Command Prompt and press Enter. This will install dependencies and might take a little while.

Then start the app with this command: node index.js (then enter)

Your terminal or command prompt should now be busy running the server.

Web browser

In your web browser, go to http://localhost:3000/ to visit your local server.

It should look a lot like our last project, because we are using the same code as a starting point.

Welcome to the workforce

Here's how your working day goes from now on:

  • Find a partner and decide who's driving and who's navigating.
  • Planning meeting: We talk about the issues in this week's sprint.
  • Choose an issue from the project wall, move it to 'Work in Progress' and write your names next to it.
  • Go to the project's Issues page on GitHub, and assign the issue to you and your partner.
  • In Github Desktop, switch to the Master branch, and click the Fetch button to see the new changes. If the button changes to 'Pull', click on it again to pull in the new changes.
  • In GitHub Desktop, now that you have the latest version, create a new branch for your work. Make sure the branch comes from the master branch, and name it something relevant to your issue, i.e. "2-submit-goes-to-a-new-page" for issue 2.
  • Go to the project's Issues page on GitHub and read the instructions for what you need to do.

Then you can work on the issue! Make a commit whenever you feel you've completed a step.

When the job is done:

  • Make sure you have committed all your changes
  • Click Publish (it might say Push instead) to save your branch back to GitHub
  • Create a Pull Request by clicking Branch then Create Pull Request in Github Desktop. Write a short comment that includes the issue number with a hashtag, i.e. "fixed #2". This lets Github know to link the Pull Request to the issue.
  • In the GitHub website, close the issue.
  • On the project wall, move the issue to the 'Done' column.
  • Next time we have a break, the mentors will do a code review. We will go over the changes with the whole class, so everyone knows what changes are coming.
  • You might be asked to make some changes first. Then your code will be merged.
  • Your changes are now in the main version of the project.

Pick up another ticket and start all over again! Remember to switch back to the master branch, fetch and pull the latest changes, and create a new branch before your next issue.
